GIT 사용

1. 계정 정보 등록
	git config -- global user.name <name>
	git config -- global user.name <사용자이메일>

2. git_prac 폴더를 git이 관리하는 폴더로 지정
	git으로 관리할 폴더로 이동
	git init

3. 기본으로 생성되는 master 브랜치를 main브랜치로 변경
	git checkout -b main : main 브랜치를 생성하고 해당 브랜치로 이동
	git branch -M main : 기본 브랜치를 main 브랜치로 지정
	git status : 상태확인

4. 파일을 생성하고 add, commit 하기
	git add <파일명> : staging area 변경된 파일들을 추가
	git commit : commit에 대한 메세지를 남기기 위해 vi에디터가 실행된다.
			i : 입력모드, esc : 입력모드에서 나오기, wq : 저장 종료

5. 파일을 마지막 commit 지점으로 되돌리기
	git restore <파일명> : 파일을 마지막 commit 상태로 rollback
	git restore --staged <파일명> staged된 파일은 unstaged 상태로 변경

6. 파일을 원하는 commit  지점으로 되돌리기
	git reset --soft <commit> : commit 내여을 디돌리고 commit만 취소(staging area에 파일이 남아있다.)
	git reset --mixed <comitt> : commit 내역을 되돌리고 staging까지 취소
	git reset --hard <comitt> : commit 내역을 되돌리고 파일을 원하는 commit지점으로 rollback
	git revert <commit> : 지정된 commit인해 발생한 변경을 되돌린다.. 새로운 commit을 생성해 기존의 commit내역에 추가한다.

7. 원격 리포지토리와 추가 및 push/pull 하기
	git remote add origin <repository 주소>
	git pull --allow-unrelated-histories origin main
		* 원격리포지토리에서 처음 pull 할 때 만
	git push -u origin main
		-u : push할 기본 원격 저장소와 브랜치를 기억
		     앞으로는 push할때 원격저장소와 브랜치를 작성할 필요가 없다.
